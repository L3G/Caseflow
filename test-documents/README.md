# Test documents

Five PDFs you can upload via **Initiate case** to exercise every path through the agent without generating your own.

| File | Client (in PDF) | What it tests |
|---|---|---|
| `first_national_bank_statement_april_2026.pdf` | Jordan A. Park | **Happy path** — reconciles ($1,200 + $5,000 − $2,400 = $3,800), means test passes ($60K annualized < $76K), case lands at `AttorneyReviewPending` |
| `sierra_pacific_statement_march_2026.pdf` | Maya R. Okonkwo | **Happy-path variant** — different client, different bank, also passes ($48K annualized) |
| `atlas_national_statement_feb_2026.pdf` | David T. Kim | **Means test FAILS** — $102K annualized > $76K threshold. Workflow still advances through every state; the Analyses panel + hero outcome chip surface the failure |
| `heartland_federal_statement_jan_2026.pdf` | Robin J. Castillo | **Reconciliation mismatch** — printed ending balance is $4,000 but begin + deposits − withdrawals = $3,800 ($200 discrepancy). `CheckArithmeticIntegrity` flags the case |
| `greenleaf_paystub_feb_2026.pdf` | (a pay stub) | **Wrong document type** — classified as `PayStub`, then `ExtractPayStub` stub returns failure. Case flagged with `Wrong type: PayStub` outcome chip |

## Using them

```bash
docker compose up --build
# Open http://localhost:8080, click "+ Initiate case", upload one of the PDFs
```

Reset state between tests:

```bash
docker compose down -v && docker compose up -d
```

## How they were generated

All five PDFs were generated by Claude Opus 4.7 from focused prompts that specified exact balance, deposit, withdrawal, and transaction values — designed to hit specific code paths in the agent (reconciliation, means test threshold, classification routing). They are not real bank statements; the institution names and account holders are fictional.
